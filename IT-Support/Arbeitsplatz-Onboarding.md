# Arbeitsplatz-Onboarding

## Ziel

Ein neuer Mitarbeiter soll am ersten Arbeitstag arbeitsfähig sein. Dazu gehören Benutzerkonto, Gerät, Software, Berechtigungen und eine kurze Nachkontrolle.

## Einordnung

Die Einrichtung eines Arbeitsplatzes ist ein typischer Service-Request. Der Prozess sollte standardisiert sein, damit nichts vergessen wird.

## Benötigte Informationen

Vor der Umsetzung müssen diese Informationen vorliegen:

- Name des Mitarbeiters
- Abteilung
- Rolle oder Tätigkeit
- Startdatum
- Vorgesetzter
- benötigte Software
- benötigte Gruppen und Berechtigungen
- benötigte Hardware
- besondere Anforderungen, z. B. Homeoffice, VPN, Telefon oder Drucker

## Planung

1. Ticket im Ticketsystem prüfen.
2. Fehlende Informationen beim Anforderer nachfragen.
3. Priorität festlegen, z. B. hoher Bedarf bei Start am nächsten Tag.
4. Prüfen, ob ein vorhandener Arbeitsplatz wiederverwendet werden kann.
5. Standardvorlagen für Benutzerkonten oder Gruppenmitgliedschaften prüfen.

## Benutzerkonto anlegen

1. `Active Directory Users and Computers` öffnen.
2. Passende OU auswählen.
3. Neuen Benutzer erstellen.
4. Anzeigename, Benutzername und E-Mail-Adresse eintragen.
5. Temporäres Passwort vergeben.
6. Option `Benutzer muss Kennwort bei der nächsten Anmeldung ändern` aktivieren.
7. Benutzer zu passenden Gruppen hinzufügen.

## Arbeitsplatz vorbereiten

1. Gerät bereitstellen.
2. Windows installieren oder vorhandenes Gerät zurücksetzen.
3. Updates installieren.
4. Gerät der Domäne beitreten lassen.
5. Benötigte Software installieren.
6. Drucker, Netzlaufwerke und Standardanwendungen prüfen.

## Funktionstest

Vor Übergabe prüfen:

- Anmeldung funktioniert
- Netzlaufwerke sind sichtbar
- wichtige Anwendungen starten
- Internet und interne Systeme sind erreichbar
- Drucker ist verfügbar
- Benutzerrechte passen zur Rolle

## Übergabe

1. Zugangsdaten sicher an den berechtigten Ansprechpartner übergeben.
2. Benutzer kurz einweisen.
3. Auf erste Passwortänderung hinweisen.
4. Ansprechpartner für Rückfragen nennen.

## Nachkontrolle

Am ersten Arbeitstag oder kurz danach aktiv nachfragen:

- Funktioniert die Anmeldung?
- Fehlt Software?
- Funktionieren Drucker und Netzlaufwerke?
- Gibt es offene Fragen?

## Abschluss im Ticket

Das Ticket wird erst geschlossen, wenn die Einrichtung geprüft und dokumentiert wurde.

Dokumentieren:

- Gerät
- Benutzerkonto
- Gruppen
- installierte Software
- durchgeführte Tests
- Übergabezeitpunkt
